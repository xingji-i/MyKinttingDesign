### 一、 核心设计理念与用户价值

在开始编码之前,先明确产品的核心价值：**它是一个可视化的、智能的编织图纸编辑器**。

-   **对用户：** 像操作Excel一样简单,但内置了专业的编织语言(符号字体)。
-   **对AI：** AI不仅要能“看懂”文字,还要能“理解”图纸(像素/格子)的结构,并具备基础的数学计算能力(尺码换算)。

### 二、 功能需求模块划分

为了清晰地推进项目,可以将功能划分为以下几个核心模块：

#### 1. 核心编辑器模块(图纸画布)
这是软件的基础,类似于Excel的网格系统。

-   **网格管理：** 创建新画布时定义初始针数(列)和行数；支持动态增加/删除行和列。
-   **单元格操作：**
    -   **字体渲染：** 能够正确加载和显示你提供的StitchMastery字体文件(如`StitchMasteryCable`系列)。这是展示麻花、交叉针等复杂符号的基础。
    -   **符号输入：** 支持从符号面板点击输入,以及直接通过键盘映射输入(例如按`!`自动变成RC符号)。
    -   **单元格合并：** 用于表示“并针”、“跨多针的麻花”等占位符号。
    -   **样式设置：** 调整字体大小、单元格背景色(用于区分不同花样区域)、边框样式。
-   **标尺与参考线：** 显示行号(左侧)和针数编号(顶部),并支持冻结窗格功能,方便浏览大图纸。

#### 2. 符号库与资源管理模块
这是你区别于普通表格软件的特色。

-   **内置字体解析器：** 程序需要自动读取你提供的PDF中,那个字体文件里每个符号对应哪个键盘字符(Unicode位点)。
-   **可视化符号面板：** 将字体中的符号按类别(2针麻花、3针麻花、加针、并针)分类显示为可点击的图标,方便用户拖拽或点击输入到画布。
-   **自定义符号：** 允许用户保存自己常用的颜色标记或组合符号。

#### 3. 项目文件管理模块
-   **文件格式：** 定义自己的项目文件格式(如`.knitproj`),用于保存画布的网格数据、单元格内容、字体格式和颜色。
-   **导入/导出：**
    -   **导出图像：** 将画布内容导出为PNG、JPG。
    -   **导出PDF：** 生成清晰的打印版图解。
    -   **导入解析：** 这是AI功能的前置基础。需要能将导入的PNG/PDF初步转换为程序内部的网格数据。

#### 4. AI智能体(Agent)模块
这是整个项目的智能核心。

-   **解析器：**
    -   **图像解析：** 利用多模态大模型,识别上传的PNG图解中的格子、符号和颜色,将其转化为结构化的数据(JSON格式的网格)。
    -   **PDF解析：** 如果是文字版PDF,直接提取文字描述和尺寸信息。
-   **推理与计算器(尺码转换)：**
    -   **逻辑：** AI需要理解编织密度的概念(即10cmx10cm有多少针多少行)。
    -   **输入：** 用户提供现有图解(S码)和自己的身材维度(胸围、衣长等)。
    -   **输出：** AI根据图解原有的密度,重新计算L码所需的起针数、加针位置和行数,并生成新的网格数据返回到编辑器。
-   **生成器：**
    -   **输入：** 用户用自然语言描述“我想要一件圆领、插肩袖、麻花图案的毛衣”。
    -   **输出：** AI结合内置的编织规则库,生成一个初步的、符合逻辑的网格图解。

#### 5. 用户界面与账户模块
-   **响应式布局：** 确保在PC大屏和手机小屏上都能正常编辑和查看。
-   **云存储：** 用户的设计稿可以保存在云端,跨设备同步。
-   **社区分享：** 允许用户将自己设计的图解匿名或公开分享给社区。

### 三、 开发路线图(先开发什么？)

建议采用**MVP(最小可行产品)**策略,分阶段推进：

**阶段一：核心编辑器 + 符号库 (基础功能)**

-   **目标：** 实现一个可用的在线Excel式网格编辑器,能够加载字体并显示符号。
-   **任务：**
    1.  搭建前端框架。
    2.  实现网格画布(基于HTML5 Canvas或可编辑的`<div>`网格)。
    3.  集成StitchMastery字体,确保符号正确显示。
    4.  开发符号面板,点击符号能在当前单元格输入对应字符。
    5.  实现基本的单元格合并、背景色修改。
-   **预计耗时：** 1-2个月(如果是全职/兼职开发)。

**阶段二：文件管理 + AI解析/生成 (核心AI功能)**

-   **目标：** 让AI能够看懂简单的图纸,并能根据描述生成简单图案。
-   **任务：**
    1.  对接多模态大模型(如GPT-4 Vision, Claude 3.5 Sonnet等),开发图像解析功能。
    2.  开发“自然语言生成图解”功能(AI需要学会输出符合你网格数据格式的JSON)。
    3.  实现导出PNG/PDF功能。
-   **预计耗时：** 1-2个月。

**阶段三：尺码计算 + 社区/云存储 (高级功能)**

-   **目标：** 解决用户的实际痛点——改尺寸。
-   **任务：**
    1.  开发尺码换算算法(基于密度)。
    2.  接入用户系统和云数据库。
-   **预计耗时：** 1个月。

### 四、 技术框架与API选型

为了兼容PC和手机端,最经济高效的选择是开发一个**Web应用**。

#### 1. 前端框架
-   **React 或 Vue 3：** 用于构建用户界面。React的生态更丰富,Vue的学习曲线较平缓。
-   **UI组件库：** 可以考虑使用Ant Design或Vant(移动端友好)来快速搭建界面。

#### 2. 画布实现 (关键决策)
这里有两种技术路线：

-   **基于Canvas：**
    -   **优点：** 绘制性能好,适合大量格子,易于实现导出图片功能。
    -   **缺点：** 难以实现类似Excel的直接点击编辑文本功能,需要自己实现一套选中和输入逻辑。
    -   **推荐框架：** Fabric.js (一个强大的Canvas库,方便操作画布上的对象)。
-   **基于DOM(HTML表格)：**
    -   **优点：** 原生支持文本输入、光标,开发简单,就像操作Excel一样。
    -   **缺点：** 当网格达到几千格时,渲染性能可能下降,导出图片需要借助`html2canvas`等库。
    -   **推荐做法：** 直接操作`<table>`。

**对于MVP,建议采用基于DOM的方式**,开发速度快,用户体验也最接近你设想的“Excel感觉”。

#### 3. 后端与AI服务
-   **后端语言：** Python(因为AI生态最好)。
-   **Web框架：** FastAPI(高性能,易用)。
-   **数据库：** PostgreSQL,用于存储用户数据和项目文件。
-   **AI模型接口：**
    -   **OpenAI API：** GPT-4o系列,具备强大的视觉能力和逻辑推理,适合解析图像和生成图解。
    -   **Claude API：** Anthropic的Claude 3.5 Sonnet在处理视觉布局和理解复杂指令方面也表现卓越。
    -   **关键点：** 你需要编写**提示词工程**,让AI明白输出的图解必须是一个符合你前端数据格式的二维数组。

#### 4. 字体处理
-   将StitchMastery字体文件(大概率是`.ttf`或`.otf`)放在项目中,通过CSS的`@font-face`引入,供整个应用使用。

### 五、 如何设计该软件的AI Agent？

Agent的设计关键在于**状态管理**和**上下文理解**。

1.  **定义数据结构：** 首先,你需要定义一个统一的“图纸”数据结构。例如：

    ```json
    {
      "projectId": "xxx",
      "gauge": { "rowsPer10cm": 28, " stitchesPer10cm": 22 },
      "dimensions": { "chest": 100, "length": 60 },
      "grid": [
        // 二维数组,代表每一行
        [{ "char": " ", "bgColor": "#FFFFFF", "merge": null }, { "char": "!", "bgColor": "#FF0000" }],
        [{ "char": "\"" , "bgColor": "#FFFFFF" }]
      ]
    }
    ```

2.  **Agent的工作流程：**
    -   **接收指令：** 用户在聊天窗口输入“根据这个图纸,帮我改成L码”。
    -   **读取上下文：** Agent获取当前编辑器的`grid`数据和`gauge`数据。
    -   **调用工具(Tool Use/Function Calling)：**
        -   **工具1：** `parse_image(image_url)`——当用户上传图片时调用。
        -   **工具2：** `resize_pattern(current_grid, old_gauge, new_gauge, target_measurements)`——这是一个核心的Python函数,负责执行数学计算和网格缩放逻辑。AI不直接计算,而是调用这个函数。
        -   **工具3：** `generate_pattern(description)`——根据描述,调用大模型生成一个新的`grid`数组。
    -   **返回结果：** Agent将计算结果(新的`grid`数组)返回给前端,前端渲染到画布上。

3.  **提示词示例：**
    > “你是一个编织专家AI。用户会给你一张图纸(图片或文本描述)或者直接提出要求。你需要分析图纸中的针法、密度和尺寸。如果需要修改尺寸,你需要调用`resize_pattern`函数,传入当前的网格和用户的新尺寸,函数会返回新的网格数据。最后,将新网格展示给用户。”

### 总结

不需要从零开始做一个像Photoshop那样复杂的软件。核心优势在于**垂直领域的符号字体**和**AI的交互能力**。

**第一步**,可以先用Web技术(HTML + CSS + JavaScript)快速搭建一个基于表格的编辑器,把字体集成进去,让它能够“画”出PDF里的那些麻花符号。这一步完成后,你就有了一个可以演示的MVP。

**第二步**,在这个基础上,接入大模型API,实现“说句话就生成图解”和“传张图就识别”这两个最吸引人的功能。
